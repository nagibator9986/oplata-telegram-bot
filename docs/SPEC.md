# Tenri Bot — спецификация сервиса

**Версия:** 1.2 · **Дата:** 2026-07-04 · **Статус:** реализовано (standalone, aiogram 3 + SQLite)

> Бот реализован как **полностью автономный сервис** в этом репозитории.
> Раздел «Django-first вариант» в конце — план миграции, если понадобится
> перенести бизнес-логику на платформу Baqsy. Интеграция с сайтом уже работает
> через REST (см. README, раздел «Связка с сайтом»).

---

## 1. Что это и зачем

Отдельный Telegram-бот-сервис, дополняющий платформу Baqsy System, — воронка «сообщества»:

1. Клиент пишет боту → приветствие → рассказ о философии Равновесия (Тенри).
2. Бот приглашает в **закрытую группу** (философия, разборы мировых компаний, кнопки).
3. При вступлении в группу **админ получает уведомление** «присоединился такой-то».
4. Бот **дожимает** клиента в личке серией касаний.
5. Бот **анкетирует** клиента и передаёт ответы админу.
6. Бот предлагает **донат** на развитие проекта.
7. Встроенный **AI-ассистент** объясняет философию и методологию разбора компаний.
8. Админ управляет всем **из самого бота**: тексты, рассылки в группу с периодичностью, анкета, лиды.

**Ключевая метрика:** конверсия «новый контакт → участник группы → заполненная анкета → донат/заявка на аудит».

## 2. Роли

| Роль | Кто | Что видит |
|---|---|---|
| **Клиент** | любой пользователь Telegram, написавший боту | воронка + AI-ассистент |
| **Участник группы** | клиент, вступивший в закрытую группу | контент группы, рассылки, кнопки |
| **Админ** | владелец (whitelist `TENRI_ADMIN_IDS`) | админ-меню: тексты, рассылки, анкета, лиды, статистика |
| **Бот в группе** | бот с правами администратора группы | публикует рассылки, отслеживает вступления/выходы |

## 3. Воронка (реализация)

Состояния: `new → welcomed → philosophy_shown → invited_to_group → joined_group →
surveying → survey_done → donation_offered → donated`, отдельно `lost`.
Переходы — только вперёд; `lost` — из любого (кнопка «Не беспокоить», `/delete_me`).

- **Приглашение в группу** — персональная одноразовая invite-ссылка
  (`createChatInviteLink(name="lead:{id}", member_limit=1, expire=30д)`) — атрибуция
  вступившего по `invite_link.name` даже без username. Fallback — статическая ссылка.
- **Вступление** (`chat_member`, JOIN_TRANSITION): уведомление админу с карточкой лида,
  приветствие в личку, событие на сайт.
- **Дожим** — `FollowUpRule`/`FollowUpTask`: серии для «не вступил» (24ч/72ч/7д),
  «не начал анкету» (24ч/96ч), «бросил анкету» (3ч/48ч). Отмена по `stop_states`,
  тихие часы 9–21 (настраиваются), обязательный opt-out.
- **Аудит по Коду Вечного Иля** (по ТЗ, реализованы Блоки 1–4) — адаптивная анкета из
  42 шагов по стадиям: `qualify` (фильтр масштаба — сотрудники, оборот; мягкий отказ
  для мелких → приглашение в закрытый клуб, аудит не запускается), `passport` (7 полей
  паспорта компании; название компании сохраняется в `Lead.company_name`),
  `audit_choice` (стандартный / углублённый), `main` (32 аналитических вопроса
  владельца/совладельца/ТОПа). Вводные тексты между блоками (`intro_key`), FSM,
  сохранение после каждого вопроса (снапшот «вопрос+ответ» в JSON), продолжение
  с места; карточка ответов админу (длинные — частями, `send_long`); событие
  с ответами на сайт.
- **Углублённый аудит (Блок 4)** — `handlers/deep_audit.py`. При выборе «Углублённый»
  ставится `Lead.deep_audit`; после завершения своей части владелец автоматически
  получает 6 персональных ссылок (`?start=aud_<token>`, `AuditParticipant`,
  токен `secrets.token_hex(8)`). Одна ссылка — один участник: чужая/использованная
  ссылка отклоняется, владельцу своя ссылка не выдаётся. Участник выбирает роль:
  Совладелец/Топ-менеджер → вопросы `stage="main"` (32), Менеджер → `stage="manager"`
  (19, seed из `MANAGER_QUESTIONS`, отдельный блок в редакторе анкеты). Ответы —
  `SurveyResponse(kind="participant")` — не пересекаются с основной анкетой; участники
  не входят в маркетинговую воронку (funnel/дожим/донаты не трогаются). Прогресс
  сохраняется, возврат — по ссылке или через /start («Продолжить анкету аудита»).
  По завершении: карточка админам (компания/роль/владелец), владельцу — прогресс
  «X из N», кнопка «📊 Статус команды». Админ-панель: карточка лида → «🔬 Аудит
  команды» (статусы ссылок, ответы участников, ➕ ссылка до 12, повторная отправка).
- **Донаты** — оффер через N часов после анкеты (настраивается), пресеты сумм,
  реквизиты с сайта (fallback — локальный текст), «Я перевёл(а)» → уведомление админу,
  одно напоминание через 7 дней.
- **AI-ассистент** — Google Gemini (основной) + OpenAI (автоматический резерв),
  слой `services/ai_providers.py`. Системный промпт = редактируемый текст бота
  (`assistant_prompt`). Защита: 30 сообщений/лид/день, глобальный дневной бюджет
  токенов, антифлуд 2 с, окно контекста 10 сообщений, ретраи 429/5xx, быстрые
  вопросы кнопками, хэндофф «Написать человеку» под каждым ответом.

## 4. Админ-панель в боте

`/admin` → меню: Статистика · Рассылки · Тексты · Анкета · Лиды · Настройки.

- **Тексты** — каждый экран бота = слот `BotText` (23 слота, seed). Правка с
  предпросмотром 1:1, фото, HTML-разметка, откат последней правки (`TextRevision`).
- **Рассылки** — мастер за 4 шага: текст/фото → кнопки (`Текст | URL`) → цель
  (группа / все в личку / сегменты: в группе, не в группе, прошли анкету, донатеры) →
  когда (сейчас / разово / ежедневно / еженедельно / ежемесячно) → предпросмотр →
  запуск. Управление: пауза/возобновление, «отправить сейчас», удаление, статистика
  (отправлено/ошибки/блокировки).
- **Анкета** — добавление (текст/число/выбор/мультивыбор/контакт), скрытие
  (soft-delete), порядок ↑↓. Ответы хранят снапшот вопроса — правки не ломают историю.
- **Лиды** — поиск (имя/username/телефон/`last`), карточка, ответы анкеты, написать
  лиду, заметки, пометить lost; кнопка «Ответить» под входящими вопросами клиентов.
- **Настройки** — тихие часы, задержка донат-оффера, вкл/выкл ассистента и дожима,
  тест-режим (рассылки только админам), статическая ссылка группы.

## 5. Надёжность

1. Всё состояние — в SQLite (WAL): рестарт не теряет расписания/дожим/прогресс анкет.
2. Планировщик — asyncio-цикл 30 с: рассылки (`next_run_at`, bump до запуска —
   без дублей), дожим, донат-офферы.
3. Лимиты Telegram: DM ≤20 msg/с, `RetryAfter` — пауза и продолжение,
   403 → `is_blocked`, журнал `message_log` (unique по broadcast+lead+run) — повторный
   запуск не дублирует.
4. Идемпотентные апсерты лидов; FSM-guard «только вперёд»; дубли апдейтов безопасны.
5. PII: `/delete_me` — анонимизация; доступ к админ-функциям — whitelist.

## 6. Интеграция с платформой Baqsy (реализована)

| Направление | Что | Как |
|---|---|---|
| бот → сайт | события new_lead / joined_group / left_group / survey_completed(+answers) / donated | `POST /api/v1/tenri/leads/sync/`, `X-Bot-Token`, fire-and-forget + ретрай |
| сайт → бот | реквизиты доната (`donation_*` ContentBlock) | `GET /api/v1/content/`, fallback — локальный слот |
| CRM | `TenriLead` + `TenriSurveyResult` в админке сайта, автопривязка к `ClientProfile` по `telegram_id` | приложение `apps/tenri_sync` в oplata-project |
| продажа | CTA «Заказать разбор компании» → `{SITE_URL}/tariffs?src=tenri` | кнопка после анкеты |

## 7. План миграции на Django-first (если понадобится)

При росте нагрузки или необходимости единой БД: перенести модели `apps/tenri`
в Django (PostgreSQL), бот превратить в тонкий клиент REST API, рассылки — на
Celery beat, FSM — на RedisStorage, long-polling — на webhook за nginx.
SQLAlchemy-слой позволяет промежуточный шаг — сменить SQLite на PostgreSQL одним URL.
