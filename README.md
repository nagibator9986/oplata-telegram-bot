# Tenri Bot — бот философии Тенри-Равновесия

Самостоятельный сервис: **aiogram 3 + SQLite**. Работает автономно, опционально
интегрируется с платформой Baqsy (Django на VPS, репозиторий `oplata-project`).
Полная спецификация: `docs/SPEC.md`.

## Что делает

**Клиентская воронка:**
`/start` → приветствие → карусель философии Равновесия → персональная ссылка в закрытую
группу → уведомление админу о вступлении → дожим в личке (3 серии касаний) → аудит по
Коду Вечного Иля (фильтр масштаба с мягким отказом → паспорт компании → выбор аудита →
32 вопроса владельца/ТОПа, всё редактируется) → ответы админу → донат-оффер →
AI-ассистент (Gemini) по методологии.

**Углублённый аудит (Блок 4 ТЗ):** если владелец выбрал «Углублённый», после его части
анкеты бот автоматически присылает ему 6 персональных ссылок для команды (2–6 участников).
Участник по ссылке выбирает роль: Совладелец / Топ-менеджер → 32 вопроса блока владельца,
Менеджер → 19 вопросов менеджерского блока. Одна ссылка — один участник; прогресс
сохраняется, возврат по той же ссылке или через /start. Каждая завершённая анкета уходит
админам (с компанией, ролью и владельцем) и владельцу — прогресс «X из N». Участники не
попадают в маркетинговую воронку: дожим и донат-офферы их не касаются.

**Админ-панель в самом боте** (`/admin`):

| Раздел | Возможности |
|---|---|
| 📝 Тексты | правка любого сообщения бота с предпросмотром и откатом, фото, HTML-разметка |
| 📣 Рассылки | мастер: текст → кнопки → цель (группа/личка/сегмент) → сейчас/разово/ежедневно/еженедельно/ежемесячно → предпросмотр; пауза, «отправить сейчас», статистика |
| 📋 Анкета | полный редактор вопросов обеих анкет (основной и менеджерской): добавить (текст, число, выбор, мультивыбор, контакт), изменить текст/варианты/обязательность, предпросмотр «как увидит клиент», порядок ↑↓ внутри своего блока, удаление с подтверждением и восстановление из «Скрытых» — прошлые ответы не теряются |
| 👥 Лиды | поиск, карточка, ответы анкеты, написать лиду, заметки, «Ответить» на вопросы клиентов; 🔬 панель аудита команды: статус по каждой ссылке, ответы участников, досоздание и повторная отправка ссылок |
| 📊 Статистика | воронка по шагам, вступления, анкеты, углублённые аудиты, донаты, AI-токены |
| 🩺 Диагностика | состояние деплоя: жив ли планировщик (свежесть тика), цепочка AI-провайдеров, очередь дожима и активные расписания, путь и размер БД, интеграция с сайтом, число админов |
| ⚙️ Настройки | тихие часы, задержка доната, вкл/выкл ассистента и дожима, тест-режим, запасная ссылка |

## Запуск

### 1. Создать бота и группу
1. @BotFather → `/newbot` → получить токен.
2. Создать закрытую группу, **добавить бота администратором** (право «Приглашать пользователей» обязательно — для персональных invite-ссылок).
3. Узнать `group_id`: добавьте в группу @getidsbot или запустите бота и посмотрите лог.

### 2. Конфигурация
```bash
cp .env.example .env   # заполнить BOT_TOKEN, GROUP_ID, ADMIN_IDS
```

### 3a. Локально
```bash
python -m venv .venv && . .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install -r requirements.txt
python main.py
```

### 3b. Docker (рекомендуется для VPS)
```bash
docker compose up -d --build
docker compose logs -f
```

### 3в. Railway
Готово из коробки: `railway.json` + `Dockerfile` + `entrypoint.sh`. Обязательно подключите
**Volume на `/data`** и задайте `TENRI_DB_PATH=/data/tenribot.db` (иначе БД сотрётся при
redeploy). Пошагово — [docs/DEPLOY_RAILWAY.md](docs/DEPLOY_RAILWAY.md).

## Архитектура

```
main.py ──► Dispatcher (aiogram 3, MemoryStorage)
   │            ├─ middlewares/lead.py   — апсерт лида на каждый апдейт
   │            ├─ handlers/admin/*      — админ-панель (whitelist ADMIN_IDS)
   │            └─ handlers/*            — клиентская воронка
   └──► services/scheduler.py — фоновый цикл (30 c):
             1) рассылки по расписанию   → services/broadcaster.py
             2) дожим-серии (FollowUpTask, тихие часы 9–21)
             3) донат-офферы и напоминания
SQLite (WAL) ◄── db/models.py + db/repo.py — всё состояние в БД,
                 рестарт процесса ничего не теряет
```

Ключевые решения:
- **Персональные одноразовые invite-ссылки** (`name="lead:{id}"`, member_limit=1, TTL 30 дней) — точная атрибуция вступивших даже без username. Fallback — статическая ссылка из настроек.
- **Дожим отменяется сам**: переход лида вперёд по воронке отменяет запланированные касания (`stop_states`); «Не беспокоить» и блокировка бота отменяют всё.
- **Ответы анкеты сохраняются после каждого вопроса** — обрыв/рестарт не теряет прогресс, продолжение с места остановки.
- **Лимиты Telegram**: DM-рассылка ≤20 msg/с, обработка `RetryAfter`, 403 → лид помечается blocked; повторный запуск рассылки не дублирует (журнал `message_log`).
- **Тест-режим**: рассылки уходят только админам — безопасная проверка перед боевой отправкой.
- Все переходы воронки — только вперёд; идемпотентные апсерты (дубли апдейтов Telegram безопасны).

## Связка с сайтом на VPS (опционально)

Бот и платформа Baqsy связаны через REST (при недоступном сайте бот работает автономно):

```
tenri-bot ──POST /api/v1/tenri/leads/sync/──► Django (VPS)   события: new_lead, joined_group,
          ──GET  /api/v1/content/──────────►                 left_group, survey_completed(+ответы), donated
```

- **Лиды и анкеты бота видны в CRM сайта** (админка → «Лиды Tenri Bot»); лид автоматически привязывается к `ClientProfile` по `telegram_id` — сквозная воронка «группа → платный аудит».
- **Реквизиты доната бот берёт с сайта** (ContentBlock `donation_*`), fallback — локальный текст.
- **CTA «Заказать разбор компании»** после анкеты ведёт на `{SITE_URL}/tariffs?src=tenri`.
- Auth — заголовок `X-Bot-Token` (= `BOT_API_SECRET` бэкенда, константное сравнение).

**Включение:**
```bash
# 1. В oplata-project (на VPS): применить миграцию приложения apps/tenri_sync
docker compose -f docker/docker-compose.yml exec web python manage.py migrate tenri_sync
# 2. Здесь, в .env:
TENRI_PLATFORM_URL=https://ваш-домен.kz
TENRI_PLATFORM_TOKEN=<BOT_API_SECRET из .env бэкенда>
```

## AI-ассистент (Google Gemini)

Основной провайдер — **Gemini** (`TENRI_GEMINI_API_KEY`, ключ бесплатно на
https://aistudio.google.com/apikey; модель по умолчанию `gemini-2.0-flash` — быстрая
и дешёвая). Опциональный резерв — OpenAI: если Gemini вернул ошибку/пустой ответ,
запрос автоматически уходит второму провайдеру (`services/ai_providers.py`).

Многоуровневая защита расходов и качества:

| Уровень | Механизм |
|---|---|
| Диалог | быстрые вопросы кнопками; под ответом — «Написать человеку» (хэндофф на живого админа) |
| Лид | лимит 30 сообщений/день, антифлуд-кулдаун 2 с, окно контекста 10 сообщений |
| Проект | глобальный дневной бюджет токенов (`TENRI_AI_DAILY_TOKEN_BUDGET`), расход виден в 📊 Статистике |
| Отказ | ретрай на 429/5xx → второй провайдер → вежливый fallback на человека |

Системный промпт («канон философии») — обычный текст бота: правится через
📝 Тексты → «Ассистент · системный промпт», без перезапуска.

## Данные и приватность

Лиды содержат PII (имя, телефон). Команда `/delete_me` анонимизирует лида.
БД — один файл `data/tenribot.db` (backup = копия файла; WAL-режим).

## Что доработать при росте (осознанные упрощения v1)

- FSM aiogram в памяти: при рестарте теряется только «положение в диалоге» (прогресс анкеты — в БД). При переходе на webhook/масштабирование — RedisStorage.
- Кнопки у текстов воронки фиксированы сидом (правится текст/фото); кнопки рассылок — полностью настраиваемые.
- Донаты v1 — реквизиты карты + «Я перевёл» (ручная сверка). CloudPayments-ссылку/Stars можно добавить в `handlers/donation.py`.
- SQLite достаточно до ~десятков тысяч лидов; дальше — PostgreSQL (SQLAlchemy позволяет сменить одним URL).
